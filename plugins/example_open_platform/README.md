# Example Open Platform Plugin

This directory shows the recommended plugin-style layout for third-party NOVA extensions.

## Suggested structure
- `adapters/` for upstream platform adapters
- `tools/` for callable tools
- `templates/` for platform, role, deploy, or scenario presets
- `docs/` for operator-facing setup notes

## Usage
Use this as a scaffold when integrating a new customer-specific capability without modifying the NOVA core tree.
