# Plugin Layout

## Goal
Allow third-party capabilities to be organized without modifying the NOVA core tree.

## Recommended layout
- `plugins/<plugin_name>/adapters`
- `plugins/<plugin_name>/tools`
- `plugins/<plugin_name>/templates`
- `plugins/<plugin_name>/docs`

## Minimum metadata
- plugin name
- version
- owner
- supported platform or capability
- config requirements
